# Redis常见面试题
## 1.Redis你们一般存哪些数据？
- 1.存放频繁访问但是不经常变更的数据
    - 用户信息
    - 功能开关和配置项
- 2.计算昂贵的查询结果：将一些复杂的、耗时的查询或计算结果缓存起来，避免重复计算。
    - 每日报表数据
·  
## 2.如何保证redis和数据库的数据一致性？
- 1.在应用层通过事务进行控制，先更新数据库，再更新redis缓存，确保两者数据一致。
- 2.使用消息队列，当数据库更新后，发送消息通知缓存更新。
- 3.设置合理的缓存过期时间，定期刷新缓存，减少数据不一致的风险。
### 如果是运维直接修改数据库的数据呢？
- 1.使用数据库触发器，在数据变更时自动发送消息通知缓存更新。
- 2.可以使用CDC工具，监听数据库的变更日志，实时同步到redis缓存。
- 3.如果修改的数据比较多，可以考虑重启Redis服务，重新加载缓存数据。

## 3. Redis为什么这么快？
- **内存存储**：Redis将数据存储在内存中，读写速度极快。
- **单线程模型**：Redis采用单线程模型，避免了多线程的上下文切换和锁竞争问题。
- **事件驱动**：Redis使用事件驱动模型，使用I/O多路复用技术，能够高效地处理大量并发连接。
- **数据结构优化**：Redis使用高效的数据结构，如跳表、哈希表等，优化了数据的存储和访问效率。
